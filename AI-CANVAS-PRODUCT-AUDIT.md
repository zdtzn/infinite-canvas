# AI 画布生图网站专业审查报告

审查日期：2026-07-21
审查版本：`fb16781`
审查范围：首页、生图工作台、提示词库、资产、无限画布、模型配置、任务生命周期、存储同步、插件、部署与代码架构。

## 结论先行

这不是一个已经适合公网多人使用的成熟产品，而是一个功能覆盖很广、工程野心很大的个人创作工具原型。最大问题不是“按钮还不够漂亮”，而是产品定位与底层架构互相冲突：前端直连模型、浏览器保存密钥和数据、任意脚本/插件执行、无账号体系和无服务端任务队列，都只适合可信用户的本地环境；一旦放到公网给朋友使用，安全、任务可靠性、数据隔离和升级兼容性会同时成为硬伤。

本次共记录 **49 项问题：16 项严重、23 项建议修改、10 项可以优化**。

## 审查依据与限制

- 已逐项阅读关键源码，包括生图任务、画布交互、状态管理、提示词来源、插件、存储、Nginx 与 Docker 部署。
- `npm run typecheck` 和 `npm run build` 均通过。
- 生产构建主 JS 为 **3,287.88 kB，gzip 后 1,049.46 kB**，Vite 明确给出大包警告。
- 公网地址 `http://118.190.159.129:3000` 在当前审查环境中无法建立连接；本地浏览器自动化也受运行环境隔离限制。因此动态视觉结论同时依据源码中的布局/响应式规则、现有功能路径和此前真实使用中出现的任务丢失、图片慢、下载慢、`Failed to fetch` 等问题。上线前仍需补一次真机桌面/移动端回归测试。

## 关键异常与边界测试清单

上线前至少要自动化覆盖以下流程：

1. 生成中切换路由、刷新、后退、关闭标签页、断网再恢复。
2. 生成按钮连续双击、十张并发、429、502、503、524、返回空 `data`、返回 URL 而非 base64。
3. Key 无效、Key 无该模型权限、模型列表可拉取但生图接口不可用。
4. 上传 0 字节、非图片伪装、超大像素、HEIC、CMYK、透明 PNG、同名文件、剪贴板权限拒绝。
5. IndexedDB 配额满、隐私模式不可写、存储被清理、Blob 丢失但元数据仍在。
6. 两个标签同时编辑同一项目、同时同步 WebDAV、删除后另一标签继续保存。
7. 500 个节点、1000 条边、100 张 4K 图片、连续缩放拖拽 10 分钟。
8. 中文输入法组合期间 Enter、文本编辑时 Delete/Ctrl+A/Ctrl+Z 不应误作用到画布。
9. 插件加载失败、插件升级中断、来源内容被篡改、脚本死循环或发起大量请求。
10. 提示词来源一个超时、一个 404、一个格式变更，其余来源应继续显示并标明健康状态。
11. 下载跨域失败、下载中断、重复文件名、批量下载部分失败。
12. 320px 手机、768px 平板、125%/200% 缩放、深浅主题、高对比度和仅键盘操作。

## 推荐目标架构

### 1. 前端

- React 路由按页面拆包；工作台、画布、CodeMirror、Agent、视频能力懒加载。
- Canvas 文档状态规范化为 `nodesById/edgesById`；选择、悬浮、视口等瞬时状态与可持久数据分离。
- React Query 只管理服务端任务/资产；IndexedDB 作为离线缓存，不再是唯一真相来源。
- 画布使用 XYFlow/React Flow 或实现空间索引与虚拟化；图片节点使用缩略图，放大时再取原图。

### 2. API/BFF

- 邀请制认证、workspace 权限、Key 加密、额度和速率限制。
- 统一模型适配器：`capabilities`、参数校验、错误映射、幂等请求、供应商故障切换。
- 禁止浏览器直接得到供应商 Key；日志自动脱敏。

### 3. 任务系统

- `jobs`、`job_outputs`、`job_events` 三层数据；Redis/BullMQ 或同等队列执行。
- 前端通过 SSE/WebSocket/轮询订阅；任何页面都能恢复任务。
- 每个输出保留 prompt、模型、参数、参考图、父版本、费用和实际尺寸。

### 4. 文件与图库

- 原图进入 OSS/S3，缩略图走 CDN；数据库保存归属、哈希、尺寸和生命周期。
- 提示词来源由定时任务抓取并快照化，避免每个用户重复访问 GitHub 和图片源。
- 支持失效源告警、重抓、内容去重和版权/来源信息。

### 5. 安全与扩展

- 公网版关闭任意脚本；插件采用签名清单、固定哈希、沙箱和权限声明。
- CSP、HTTPS、HSTS、限流、健康检查、错误追踪和审计日志成为发布门槛。

## 可形成的差异化

最值得押注的不是复制所有竞品功能，而是：**面向中文电商团队的私有化多模型视觉工作流**。

- 多模型统一接入，但由能力 schema 自动适配，不让用户手写脚本。
- 主图工作流内置“商品主体图 → 场景变体 → 可编辑卖点排版 → 平台检查 → SKU 批量导出”。
- 画布保存完整生成血缘，任何结果都能追溯、重跑和替换模型。
- 邀请制私有空间，产品图、Key、模板与品牌资产不外泄。

## 最值得优先修改的 10 个地方

1. **上 HTTPS，并把 API Key 与模型请求移到后端 BFF。**
2. **建立持久化任务队列和全局任务中心，彻底解决切页/刷新丢任务。**
3. **增加邀请制登录、用户/空间隔离、配额和审计。**
4. **公网版禁用任意脚本和未签名插件，重做扩展安全边界。**
5. **建立模型能力 schema，按模型动态展示尺寸、参考图和专业参数。**
6. **提示词库改为服务端抓取、缓存缩略图和分页 API。**
7. **路由拆包并治理 1 MB gzip 主包，设置性能预算。**
8. **重构画布引擎：虚拟化、对齐、吸附、自动排列、图层和锁定。**
9. **电商主图改为“AI 无字底图 + 可编辑文字模板 + SKU 批量导出”。**
10. **拆分 2778 行画布控制器并补齐单测、E2E、异常网络和大画布性能测试。**

建议执行顺序是：先完成 1-4 的安全与可靠性底座，再做 5-7 的体验和性能，最后推进 8-10 的专业化。继续直接增加表面功能，会让维护成本增长得比产品价值更快。
